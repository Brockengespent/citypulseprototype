# CityPulse — прототип дашборда

Прототип демонстрационного веб-приложения для информационной системы CityPulse — городской платформы мониторинга и прогнозирования (вариант №2, дисциплина «ПрИС», ИТМО).

Прототип создан в рамках практической работы №6 с использованием vibe-coding (генерация кода по описаниям через большую языковую модель). Серверная часть упрощена до статичного слоя данных в JavaScript (mock), что соответствует ER-схеме из предыдущей работы №7.

## Что умеет прототип

- Показывает карточки районов города (Центральный, Адмиралтейский, Петроградский, Васильевский, Калининский).
- В каждой карточке — агрегированные показатели: количество онлайн-источников, средняя скорость транспорта, расход воды ЖКХ, инциденты безопасности; цвет значения сигнализирует о норме / отклонении.
- При клике на карточку открывается детализация по району: таблица источников (датчики, протоколы, последние значения, статус) и два графика (трафик и водопотребление за 24 часа).
- Кнопка «Получить прогноз» строит наивный квантильный прогноз (P10/P50/P90) по выбранному показателю и горизонту 1–24 часа, показывает ожидаемую MAPE.

## Технологический стек

- Vanilla HTML/CSS/JavaScript (без сборщиков).
- [Chart.js 4.4](https://www.chartjs.org/) подгружается с CDN.
- Слой данных — JavaScript-объекты, сгенерированные при загрузке (mock-data.js), повторяет схему `District → CityObject → DataSource → Measurement` из ER-диаграммы.

## Структура файлов

```
citypulse-prototype/
├── index.html       — разметка страницы и стили
├── mock-data.js     — слой данных: районы, источники, измерения, функция прогноза
├── app.js           — UI-логика: рендер карточек, таблицы, графики, прогноз
└── README.md        — этот файл
```

## Как запустить локально

Достаточно открыть `index.html` в современном браузере (Chrome, Firefox, Safari, Edge). Сервер не требуется.

Для надёжности (некоторые браузеры блокируют локальные подключения скриптов через `file://`) можно поднять простой статический сервер:

```bash
# из папки проекта
python3 -m http.server 8000
# затем открыть http://localhost:8000
```

## Деплой на GitHub Pages

1. Запушить репозиторий на GitHub.
2. Settings → Pages → Source → выбрать ветку `main`, папка `/ (root)`.
3. Сохранить — через 1–2 минуты прототип станет доступен по адресу
   `https://<ваш-логин>.github.io/<имя-репо>/`.

## Ограничения прототипа

- Данные сгенерированы случайно при загрузке страницы — реальной интеграции с источниками нет.
- Прогноз — наивный baseline (взвешенное скользящее среднее), не ML-модель.
- Аутентификация и авторизация не реализованы (предполагаются на следующей итерации).
- Прототип не тестировался автоматически (нет unit/e2e тестов).
